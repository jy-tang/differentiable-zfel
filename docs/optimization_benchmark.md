#### Notes

- This run uses a seeded 8 GeV, 9.8 keV case with resonant `K0` and a fixed initial bucket.
- The optimizer starts from exactly no taper and uses only gradient-based updates.
- Current best run with the 3-parameter physical taper model:
  start position + linear slope + quadratic slope.
- Seed power is `1e6 W`, no-taper final power is `12.796 GW`, and the best gradient-optimized taper reached `152.311 GW`.
- Final power gain vs no taper: `11.903x`.
- Best iterate was `36`, with `K` tapering from `1.461` down to `1.347`.
- The optimizer can overshoot later, so the script records and plots the best iterate rather than the last iterate.
- Follow-up run with tiny nonzero initial taper:
  this was added so taper start would get gradient signal immediately instead of starting from an exactly flat profile.
- Follow-up run result:
  best iterate `9`, final power `136.182 GW`, power gain `10.642x`, with `K` tapering from `1.461` down to `1.394`.
- Follow-up run interpretation:
  the tiny initial taper improved convergence speed and still found a `>10x` solution, but it did not materially move the taper earlier.
- Follow-up run interpretation:
  the fitted taper start stayed late at about `start_frac = 0.373`, compared with about `0.383` in the earlier `11.9x` run.
- Option 2 test:
  I changed the parameterization to decouple taper start from taper strength by using `start_frac`, end amplitude, and curvature as the 3 control variables.
- Option 2 result:
  this version underperformed badly in the current tuning, reaching only `1.162x` final power gain, with a fitted `start_frac = 0.432` and a very small learned taper amplitude.
- Option 2 lesson:
  decoupling start from amplitude is not enough by itself. In the current objective and optimizer tuning, the model collapses toward a weak-taper solution instead of learning an earlier strong taper.
- Current working configuration:
  reverted to the previous 3-parameter model `start + linear + quadratic`, which is the latest usable setup and still supports `>10x` gain.
- Flexible-control follow-up:
  I added a separate high-dimensional taper model in `scripts/seeded_gradient_flexible.py` with one start parameter, one end-amplitude parameter, and `8` monotone shape controls.
- Flexible-control result:
  the first run reached only `29.927 GW`, or `2.339x` over the `12.796 GW` no-taper baseline.
- Flexible-control change:
  the new model uses a monotone interpolated taper shape so gradient-based optimization has more control degrees of freedom than the 3-parameter model.
- Flexible-control lesson:
  more control parameters alone do not automatically help. With the current objective and initialization, the extra shape freedom makes the optimization problem harder and underperforms the simpler 3-parameter taper.
- Flexible-control next step:
  warm-start the flexible shape from the successful quadratic taper or regularize the shape around that baseline instead of starting from a nearly flat generic shape.
- Flexible warm-start follow-up:
  I warm-started the high-dimensional taper from the successful `>10x` quadratic solution instead of a generic shape.
- Flexible warm-start result:
  the best iterate was the very first one, with `137.836 GW` and `10.771x` gain over no taper.
- Flexible warm-start lesson:
  the extra shape controls can preserve a strong solution when initialized well, but in this run they did not improve beyond the simpler quadratic taper.
- Flexible warm-start lesson:
  this suggests the main bottleneck is no longer parameterization flexibility alone; the objective or optimizer needs to actively exploit the added degrees of freedom, otherwise the warm start is already near a local optimum.
- Lessons learned:
  the big gain came only after enforcing an exact no-taper start, while keeping nonzero gradient signal.
- Lessons learned:
  using `softplus(theta) - log(2)` for the taper coefficients made `theta = 0` correspond exactly to zero taper; the earlier version accidentally started from a nonzero taper and gave misleading behavior.
- Lessons learned:
  the simple 3-parameter physical model worked once it was allowed to explore a stronger taper range; the earlier conservative coefficient scales kept the optimizer in a weak-taper regime.
- Lessons learned:
  optimizing primarily for final power worked better here than mixing in extra pulse-energy weighting.
- Lessons learned:
  weaker lower-`K` and smoothness penalties were necessary; overly strong regularization prevented the optimizer from reaching the high-gain seeded taper regime.
- Process note:
  after each run, record the figures, key metrics, what changed, and the main lesson here before moving on to the next idea.

#### Notes

- Configuration:
  same 3-parameter `start + linear + quadratic` taper model as the successful 1 MW run, but with seed power changed from `1e6 W` to `2e9 W`.
- Result:
  no-taper baseline increased to `18.816 GW`, but the best optimized result was only `30.942 GW`, or `1.644x`, at iteration `2`.
- Change from previous run:
  this isolates seed power as the main change; the taper parameterization and gradient optimizer stayed the same.
- Lesson:
  the taper strategy that works well at `1e6 W` does not transfer directly to `2e9 W`. The stronger seed changes the optimization landscape enough that the run rises briefly and then collapses.
- Lesson:
  for the 2 GW seeded case, the current objective and learning-rate setup are too aggressive or misaligned; they do not preserve the early improvement.
- Next step:
  retune the objective or step size specifically for the high-seed regime before drawing conclusions from this case.
- Oscillation-penalty follow-up:
  I added a differentiable post-saturation oscillation penalty to the objective so the optimizer would explicitly dislike power dips after saturation.
- Oscillation-penalty result:
  the best iterate remained the same at `30.942 GW`, or `1.644x`, and the later collapse still occurred.
- Oscillation-penalty lesson:
  this penalty by itself is not strong enough or not activated early enough to change the optimization path in the 2 GW regime.
- Oscillation-penalty lesson:
  the problem in the strong-seed case is more fundamental than just missing a smoothness term after saturation; the whole taper strategy likely needs retuning for that regime.
- Early-start retune follow-up:
  I changed the taper-start search window to favor earlier starts and reduced the learning rate for the `2 GW` seeded case.
- Early-start retune result:
  the best run reached `35.535 GW`, or `1.889x` over the `18.816 GW` no-taper baseline, which is better than the previous `1.644x`.
- Early-start retune result:
  the optimized taper start moved to `start_frac = 0.085`, much earlier than the previous `~0.371`.
- Early-start retune lesson:
  for the strong-seed regime, the optimizer was not refusing early taper on physics grounds; it was being held back by the parameterization window and optimization dynamics.
- Early-start retune lesson:
  forcing the start search earlier and using a smaller step size produces a materially earlier taper and a better result, even though the overall gain is still far below the `1 MW` case.
- Negative-slope penalty follow-up:
  I changed the oscillation term to penalize negative post-saturation log-power slope directly, instead of squaring the negative slope.
- Negative-slope penalty result:
  the latest run regressed to `23.218 GW`, or `1.234x` over the `18.816 GW` no-taper baseline, with best iterate `9`.
- Negative-slope penalty result:
  the fitted taper remained shallow, with `K` only moving from `1.461` down to `1.458`.
- Negative-slope penalty change from previous run:
  the seed power, 3-parameter taper model, and early-start search window stayed the same; the only intended objective change was the oscillation term shape.
- Negative-slope penalty lesson:
  penalizing negative post-saturation slope in this simple form is too restrictive or poorly aligned with the strong-seed objective, and it suppresses useful taper strength instead of guiding an earlier, better taper.
- Negative-slope penalty lesson:
  the current best-known `2 GW` result is still the earlier-start retune at `35.535 GW` and `1.889x`; this latest option-2 run is a regression, not an improvement.
- Next step:
  keep the earlier-start retune as the strong-seed baseline and, if oscillation control is still needed, target the post-saturation region more explicitly without weakening the overall taper amplitude.

#### Notes

- This run used a lightweight recurring benchmark configuration: `npart=64`, `s_steps=16`, `z_steps=16`, 6 taper controls, 5 gradient steps, and 8 calls for each derivative-free baseline.
- Deterministic best pulse-energy ratio vs no taper:
  `Gradient` 1.023x, `Powell` 1.128x, `Random` 1.193x.
- Held-out stochastic mean ratio vs no taper:
  `Gradient` 1.289x, `Powell` 1.323x, `Random` 1.242x.
- The hand reference taper was not a strong baseline in this reduced setting: 0.433x deterministic and 0.972x held-out mean.
- Interpretation:
  this first automated benchmark proves the workflow, but not yet the final scientific claim. The next stronger comparison should increase resolution and tune budgets so gradient information has enough room to outperform derivative-free search.
- The strongest current positive signal is robustness:
  `Gradient` beat `Random search` on held-out mean with fewer evaluations, and its held-out robust score `mean - 0.5 x std` is slightly better than `Powell` because Powell bought extra mean with noticeably higher shot-to-shot spread.
- Keep the comparison fair by matching the control parameterization and reporting either simulator-call budgets or wall-clock time.
- For stochastic SASE studies, always report held-out mean and spread on an explicit noise batch rather than only the training objective.
